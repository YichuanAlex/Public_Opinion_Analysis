## 一、这个工具是干什么的

这是一个帮助团队判断**「一条小红书笔记值不值得花钱加热」**的本地工具。

它会学习我们自己历史投放数据里的规律——哪类内容、哪种互动结构，最终 CPE 比较好——然后对你准备加热的新帖子给出预测和建议。

**核心输出：**

| 输出项 | 说明 |
|--------|------|
| 加热建议 | 推荐加热 / 建议小额测试 / 不建议加热 |
| 预测 CPE | 预估每次互动成本（≤ 1.4 算成功） |
| 建议测试金额 | 首轮投放区间 |
| 判断依据 | 哪些信号正向、哪些负向 |
| 相似历史案例 | 最像这条帖子的历史投放，附真实 CPE |

> ⚠️ 工具给的是参考建议，不是绝对结论。建议先小额跑 12–24 小时，再根据真实 CPE 决定是否加码。

## 二、使用前的准备

### 2.1 安装 Node.js（只需做一次）

Node.js 是运行本工具需要的基础程序，免费、安全。

1. 浏览器访问 **https://nodejs.org**
2. 点击绿色大按钮 **「LTS 推荐版本」** 下载安装包
3. 双击安装，全程点「继续」「安装」，无需修改任何设置
4. 安装完成后无需重启电脑

> 截图占位：nodejs.org 首页，箭头指向 LTS 下载按钮

### 2.2 获取工具文件夹

向负责人索取工具压缩包（ZIP），收到后双击解压到桌面或文档文件夹。

解压后文件夹里应包含：

```
start_tool.command   ← 双击此文件即可启动
server.js
index.html
app.js
styles.css
...
```

## 三、启动工具

**每次使用前都需要启动一次。**

1. 双击 **`start_tool.command`**
2. 首次运行 macOS 可能弹出「无法打开来自未知开发者的文件」提示：
   - 右键点击该文件 → 选「**打开**」 → 再点「**打开**」
   - 此提示只出现一次，之后直接双击即可
3. 电脑自动打开一个**黑色终端窗口**，稍等 2–3 秒浏览器会自动打开工具页面（地址：`http://localhost:5173`）

> 截图占位：工具浏览器界面全貌

> ⚠️ **黑色终端窗口不要关**，关了工具就停了。用完后关掉终端窗口即可退出。

## 四、配置 Cookie（抓取帖子信息）

Cookie 是你登录小红书的凭证，工具需要它才能通过链接自动抓取帖子的标题、正文、封面和当前互动数据。**配置一次会记住，大约每隔几周失效一次，失效后重新配置即可。**

### 4.1 从 Chrome 获取 Cookie

1. 打开 **Google Chrome**，访问 `www.xiaohongshu.com`，确认已登录
2. 按 **F12**（或右键页面 → 检查）打开开发者工具
3. 点击顶部标签栏的 **「Network（网络）」**

> 截图占位：Chrome 开发者工具，箭头指向 Network 标签

4. 刷新页面（F5）
5. 在左侧请求列表中，点击任意一条 `www.xiaohongshu.com` 开头的请求
6. 右侧面板滚动找到 **「Request Headers（请求标头）」** → 找到 **「Cookie」** 行 → 右键复制整个值

> 截图占位：Request Headers 区域，箭头指向 Cookie 行

### 4.2 粘贴到工具

1. 回到工具页面，左侧找到 **「Cookie 设置」** 折叠块，点击展开
2. 将复制的 Cookie 值粘贴进文本框
3. 点击 **「保存」**
4. 角标变为绿色 **「已配置」** 即成功

> 截图占位：Cookie 设置面板，已配置状态（绿色角标）

## 五、配置 LLM（AI 加热判断）

工具叠加了一层 AI 大模型来做综合判断，建议使用**公司内部 LLM Proxy 平台**，免费且无需申请外部 API Key。

### 5.1 申请 LLM Proxy 的 Virtual Key

1. 访问 **http://llm-proxy.intra.xiaojukeji.com/ui/**（内网）
2. 参考《大模型使用指南》开通访问权限
3. 选择 **「Virtual Key」**，填写：
   - **Owned by**：选你自己和所属 T1 团队（必填）
   - **Key 名称**：自定义，如 `heat-judge`
   - **Key 类型**：`all team models` / `default`
4. 点击 **「Create New Key」**，**立即保存 Key Secret**（只显示一次）

> 截图占位：Virtual Key 创建界面

### 5.2 查看可用模型名

在 LLM Proxy 平台选择自己的团队，进入 **「Models」** 页面，记下一个可用的模型名（如 `deepseek-r1-distill-qwen-32b`）。

> 截图占位：可用模型列表

### 5.3 在工具里填写 LLM 配置

1. 工具左侧找到 **「LLM 设置」** 折叠块，点击展开
2. 按以下填写：

| 字段 | 填写内容 |
|------|----------|
| **服务商** | 选 `OpenAI 兼容` |
| **API Key** | 上一步保存的 Virtual Key Secret |
| **Base URL** | `https://llm-proxy.intra.xiaojukeji.com` |
| **模型名** | 你查到的模型名，如 `deepseek-r1-distill-qwen-32b` |

3. 点击 **「保存」**，角标变绿即配置成功
4. 配置会持久化保存，重启工具无需重填

> 截图占位：LLM 设置面板，填写完毕状态

> 💡 也可以使用外部模型（DeepSeek / OpenAI / Gemini），在设置里切换服务商并填入对应 API Key 即可。外部 API Key 有费用，优先用内部 Proxy。

## 六、导入历史数据，训练模型

工具需要加载历史投放数据才能学习规律。**每次重新打开工具都需要重新导入一次**（数据保存在本地浏览器，不上传）。

### 6.1 准备数据

从飞书/金山表格打开历史投放数据汇总表。工具会自动识别以下关键列：

| 必须有 | 建议有 |
|--------|--------|
| 标题 | 正文 / 内容 |
| 综合 CPE（或总投放金额 + 总互动量） | 投前互动量 |

### 6.2 导入方式

**方法一：复制粘贴（推荐）**

1. 在表格里全选数据（含表头）→ 复制（`Ctrl+C` / `⌘C`）
2. 粘贴到工具左侧文本框
3. 点击 **「学习历史数据」**

**方法二：上传文件**

1. 将表格另存为 CSV
2. 点击左侧「上传 CSV / TSV / TXT」区域，选择文件
3. 点击 **「学习历史数据」**

> 截图占位：导入面板，数据已粘贴状态

### 6.3 确认训练成功

训练完成后，左侧「模型状态」显示有效样本数、成功样本数、自动 CPE 阈值，右上角变为绿色 **「已训练 XXX 条」**。

> 截图占位：训练成功后的模型状态区域

## 七、判断一条帖子

### 7.1 自动抓取帖子信息（推荐）

1. 在小红书 App 里打开要判断的帖子 → 点「···」→「**复制链接**」
2. 回到工具，右侧顶部「粘贴笔记链接」框里粘贴链接
3. 点击 **「自动获取」**
4. 稍等几秒，标题、正文、封面图、当前点赞/收藏/评论/分享会自动填好

> 截图占位：自动获取成功后的表单，各字段已填入

> 如果提示「抓取失败」，多半是 Cookie 失效了，参考第四章重新配置 Cookie。

### 7.2 手动检查并补充

自动获取后快速核对：

- **标题和正文**是否完整（偶有截断，可手动补充）
- **当前总互动量**是否正确（工具会自动求和，一般无需改）
- 封面未自动识别时，可手动点击封面区域上传图片

### 7.3 点击「判断」

确认无误后，点 **「判断」** 按钮，右侧显示 AI 综合判断结果。

> 截图占位：判断结果卡片

## 八、看懂判断结果

### 8.1 主结论卡片

| 显示内容 | 含义 |
|----------|------|
| **推荐加热** | 信号积极，历史相似样本表现好，建议正常投入 |
| **建议小额测试** | 信号存在但不稳定，先小额验证 CPE |
| **不建议加热** | 信号负向，相似历史 CPE 普遍偏高，建议跳过 |
| **预测 CPE** | 模型预测的每次互动成本（≤ 1.4 为目标线） |
| **建议金额** | 首轮测试的推荐投放区间 |
| **置信度** | 模型把握程度，低于 60% 时建议小额验证 |

### 8.2 判断依据

点击下方 **「判断依据」** 标签，查看各信号的具体分析：关键词匹配情况、当前互动结构、封面评分等。

### 8.3 相似历史

点击 **「相似历史」** 标签，查看最像这条帖子的历史案例，以及它们的真实 CPE 和投放结果。

## 九、每月追加新数据，持续学习

模型不会自动进化，它每次导入时从零重训。**持续学习 = 每月把新完成的加热数据追加进训练集**。

左侧有专门的「**＋ 追加本月新加热数据**」面板，不需要手工合并 Excel。

### 操作步骤

1. 在飞书/金山表格，把当月「加热汇总」tab **导出为 CSV**
2. 工具左侧点开 **「＋ 追加本月新加热数据」**
3. 上传 CSV 或直接粘贴表格内容
4. 点击 **「追加并重训」**

工具会自动：

- 按**发布日期**推断加热渠道（4月28日起为「薯条」，3月1日至4月27日为「合作广场」，之前为「口碑通」）
- 从**链接**提取笔记ID
- 按笔记ID/链接**去重**（同一条新数据会覆盖旧的，方便修正 CPE）
- 并入现有训练集并**立即重训**

5. 成功后显示：「**已并入：新增 X 条，更新 Y 条，去重后共 Z 条。模型已重训。**」

> 截图占位：追加成功后的提示信息

> 💡 追加不影响原有训练数据，只是扩充。训练集会越滚越大，判断越来越准。追加完成后可全选左侧文本框内容复制备份。

## 十、常见问题

**Q：工具打开后是空白页 / 加载失败？**
A：确认黑色终端窗口还开着。关掉浏览器标签，重新访问 `http://localhost:5173`。

**Q：点「自动获取」提示「无法连接」？**
A：终端窗口可能关了。重新双击 `start_tool.command` 启动，再试。

**Q：点「自动获取」提示「404 错误页」或「token 已过期」？**
A：链接里的 token 过期了。重新在 App 里打开帖子，重新复制链接粘贴。

**Q：Cookie 失效了怎么知道？**
A：之前能抓的帖子突然抓不到，或左侧 Cookie 角标变橙色，就需要重新配置（参考第四章）。一般几周失效一次。

**Q：LLM 没有给出结论，只显示本地预测？**
A：LLM 设置未配置或 API Key 已失效。参考第五章重新配置。内网访问需确认在滴滴内网或 VPN 环境下。

**Q：每次打开都要重新导入数据吗？**
A：是的，工具关闭后模型不保留。建议把历史训练数据 CSV 存放在工具同一文件夹，下次直接上传；或先做第九章的「追加」操作。

**Q：工具用完怎么关？**
A：关掉黑色终端窗口即可，浏览器标签正常关。

**Q：可以多人同时用同一份工具吗？**
A：每人在自己电脑上运行工具互不干扰。如需共享历史数据，把训练 CSV 发给对方导入即可；LLM Proxy 的 Key 建议每人单独申请。
